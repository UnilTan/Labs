# ИНСТРУКЦИЯ ПО ЗАВЕРШЕНИЮ ЛР-14

## ШАГ 1: ВЫПОЛНЕНИЕ SQL СКРИПТОВ
1. Откройте SQL Server Management Studio (SSMS)
2. Подключитесь к серверу `(localdb)\MSSQLLocalDB`
3. Выполните скрипт из файла `Database/UpdateDatabaseForLab14.sql`
4. Выполните скрипт из файла `Database/LoadImagesIntoDB.sql`

## ШАГ 2: СБОРКА И ЗАПУСК ПРОЕКТА
```bash
cd MilkProductsBinding
dotnet build
dotnet run
```

## ШАГ 3: ТЕСТИРОВАНИЕ ЧАСТИ 1 (ФАЙЛОВАЯ СИСТЕМА)
1. Запустите приложение
2. Откройте окно "ЛР-14" из главного меню
3. Проверьте:
   - DataGrid отображает продукты с изображениями (moloko.png, kefir.png, etc.)
   - ListView показывает каталог в виде плиток
   - Продукты без изображений показывают заглушку picture.jpg
   - Вычисляемое свойство pathFoto работает корректно

## ШАГ 4: ТЕСТИРОВАНИЕ ЧАСТИ 2 (ИЗОБРАЖЕНИЯ В БД)
1. Запустите проект MilkProductsImages
2. Проверьте подключение к БД
3. Используйте кнопку "Загрузить все изображения из папки" для массовой загрузки
4. Или загружайте изображения по одному через "Загрузить изображение"
5. Проверьте отображение изображений из БД в DataGrid
6. Протестируйте экспорт изображений на рабочий стол

## ШАГ 5: СОЗДАНИЕ ОТЧЕТА
1. Сделайте скриншоты:
   - DataGrid с изображениями из файловой системы
   - ListView каталог товаров
   - DataGrid с изображениями из БД
   - Процесс загрузки изображения в БД
2. Заполните отчет в файле `ОТЧЕТ_ЛР14.md`
3. Добавьте скриншоты в отчет

## ТИПОВЫЕ ОШИБКИ
1. **Изображения не отображаются** - проверьте путь к папке Images в Debug
2. **Ошибка подключения к БД** - проверьте строку подключения
3. **Заглушка не работает** - убедитесь что picture.jpg существует
4. **Проблемы с varbinary** - проверьте тип данных в БД
5. **Массовая загрузка не работает** - проверьте путь к папке с изображениями
